# ТЗ: Портативный транскрибер на Faster-Whisper XXL с GUI

Проект для Claude Code (или другой нейросети). Контекст: пользователь — психолог,
транскрибирует записи консультаций (русский язык, обычно 2 спикера, ~1 час).
Все решения ниже уже обсуждены и приняты — реализовывать по этому ТЗ.

## Цель

Портативная папка (копируется целиком на любой Windows-ПК, ничего не
устанавливает в систему) с простым GUI, которая превращает аудио/видео-записи
в размеченные транскрипты: txt (таймкоды+спикеры), srt, md.

## Ключевые решения (не пересматривать без запроса пользователя)

1. **ASR — Faster-Whisper XXL (standalone от Purfview)**, НЕ GigaAM.
   Пользователь сравнил оба и выбрал XXL. Репозиторий:
   github.com/Purfview/whisper-standalone-win. Это готовый exe без Python —
   наша задача только GUI-обёртка и оркестрация.
2. **Диаризация — встроенная в XXL**: `--diarize pyannote_v3.1 --num_speakers N`
   (+ `--min_speakers/--max_speakers`, `--diarize_device cuda|cpu`).
   Отдельный pyannote/Python НЕ нужен. Модели кешировать ВНУТРЬ портативной
   папки, чтобы ехали вместе с ней (проверить при сборке, нужен ли HF-токен
   при первом скачивании; на машинах пользователей токен требоваться не должен).
3. **Спикеры — только нейтральные «Спикер 1/2/...»**. Никаких попыток угадать,
   кто психолог. Исключение: в режиме двух дорожек OBS дорожка микрофона
   детерминированно = владелец записи, там можно подписывать.
4. **CUDA и CPU в одной сборке**: автоопределение при старте (nvidia-smi,
   проверка VRAM >= ~4 ГБ для large-модели) с ручным переключателем в GUI.
   На GT 1030 (2 ГБ) и без NVIDIA — CPU.
5. **Модель Whisper: large-v3-turbo ПО УМОЛЧАНИЮ** (выбор пользователя —
   turbo ему хватает). В сборку предзагружен ТОЛЬКО turbo (~1.6 ГБ);
   остальные модели (large-v3 и др.) в списке помечены «скачается при
   первом использовании (~N ГБ)» и докачиваются XXL по требованию в models\
   внутри папки (дальше переносятся вместе со сборкой). GUI показывает,
   какие модели уже на диске. Язык ru по умолчанию.

## Режимы обработки (автоопределение через ffprobe + возможность выбрать вручную)

- **Две+ аудиодорожки** (MKV/MKA из OBS): извлечь каждую дорожку в WAV 16k
  моно → распознать каждую отдельно (диаризация НЕ нужна — спикер известен
  по номеру дорожки) → склейка реплик по таймкодам.
- **Стерео с разведёнными каналами** — ТОЛЬКО ручной режим (пункт в списке
  режимов, автоматика его никогда не предлагает и вопросов про стерео не
  задаёт). Нужен не для OBS (там штатный мультитрек), а для чужих файлов:
  аппаратные рекордеры звонков и Android-записывалки часто пишут L = один
  голос, R = другой. channelsplit → далее как с дорожками.
- **Моно / обычная запись** (диктофон, а также любое обычное стерео): XXL с
  `--diarize pyannote_v3.1`, поле «число спикеров» в GUI (обязательно
  передавать, если известно).

### Инспекция файла и сопоставление дорожек (дизайн принят)

- Ничего не предполагаем о файле заранее: при добавлении GUI прогоняет ffprobe
  и показывает карточку: сколько аудиодорожек, кодек/каналы каждой, названия
  дорожек из метаданных (OBS умеет их прописывать).
- Предложение режима: 2+ дорожки -> режим дорожек; всё остальное (моно и
  стерео) -> диаризация. Разделение стерео по каналам — только ручной выбор.
- Таблица сопоставления: «Дорожка N -> Спикер M / исключить» (лишние дорожки,
  например с системными звуками, исключаются из обработки). Сопоставление
  сохраняется как дефолт и применяется к следующим файлам автоматически;
  при наличии имён дорожек в метаданных — сопоставлять по именам, а не номерам.

## Склейка (референс — «Пайплайн транскрибации с диаризацией.md» рядом)

- Реплики сортируются по времени начала.
- Если начало реплики попадает внутрь интервала предыдущей — пометка
  «(одновременно)» после имени спикера.
- Подряд идущие реплики одного спикера склеиваются.
- Для диаризационного режима XXL сам выдаёт спикеров в SRT/JSON — парсить его
  вывод (проверить формат: --diarize_dump, sentence/word json).

## Выходные файлы (форматы — чекбоксы в GUI)

Папка вывода — переключатель в GUI (на обеих вкладках, запоминается):
«Рядом с исходником» (по умолчанию) / «В папку...» (выбор постоянной папки;
удобно указать input\ конвертера subtitle_to_MD пользователя — тогда цепочка
транскрибер -> конвертер работает без ручного перекладывания файлов).

Референс форматов — экспорт GigaAMGUI, который понравился пользователю
(образцы: D:\Temp\_Transcription\GigaAM\Ал.Buk 2026-07-16*).
Таймкоды: ВСЕГДА полные ЧЧ:ММ:СС (например [00:03:28 - 00:03:43]) во всех
форматах — решение пользователя.

1. `_diarize_timecodes.txt` (ГЛАВНЫЙ — его ест конвертер пользователя
   subtitle_to_MD): `[ММ:СС - ММ:СС] Спикер №N: текст` построчно,
   по фразам.
2. `_diarize.txt`: блоки `[Спикер №N]` + фразы обычными строками,
   без таймкодов.
3. `.md`: сгруппировано по ходам спикеров, НО (правка пользователя к
   формату GigaAMGUI): один таймкод НАЧАЛА хода в заголовке, вся речь
   хода — слитым абзацем без разбивки на фразы-пункты:
   ```
   ### Спикер №2 — `03:28`

   Как бы много задач... (весь ход одним абзацем до смены говорящего)
   ```
4. `.srt` (опционально): субтитры, «Спикер №N: текст».

Статистику речи (шапку «# Расшифровка» с процентами) НЕ делать — её делает
конвертер пользователя из формата №1.
- Переименование «Спикер 1» → имена делает ОТДЕЛЬНЫЙ существующий инструмент
  пользователя (D:\Temp\__Codex\subtitle_to_MD\ + его gui.ps1) — не дублировать.

## Фича: «Файл для прослушивания» (ОТДЕЛЬНАЯ ВКЛАДКА GUI, этап 4)

Самостоятельный мини-конвертер, полностью независимый от транскрибации:
своя вкладка со своим списком файлов (drag&drop, любые аудио/видео форматы)
и кнопкой «Создать». Можно открыть GUI только ради него.

Поведение по входу (через ffprobe): мультитрек -> нормализация каждой
выбранной дорожки + сведение; обычный файл (моно/стерео любого формата) ->
нормализация + перекодирование в выбранный пресет (заодно решает задачу
«ужать большой WAV в opus»). Результат рядом с исходником,
суффикс « (прослушивание)». Параметры:

- **Формат**: Opus (по умолчанию) / MP3.
- **Качество** — три пресета, битрейт подставляется под формат
  (никакого свободного ввода цифр):
  Компактный (Opus 32 / MP3 64), Стандарт — по умолчанию (Opus 64 / MP3 128),
  Высокий (Opus 96 / MP3 192).
- **Каналы**: Моно-сведение (по умолчанию) / Стерео «голоса по ушам»
  (дорожка 1 -> L, дорожка 2 -> R).
- **Нормализация громкости**: галочка, ВКЛЮЧЕНА по умолчанию — ffmpeg loudnorm
  на каждую дорожку до сведения (дорожки OBS почти всегда разной громкости).
- Частота дискретизации автоматически (48k Opus / 44.1k MP3); скорость и
  шумодав НЕ делать. Исключённые дорожки (системные звуки) не попадают.

## GUI

- PyQt6 или аналог; очередь файлов drag&drop, выбор режима (авто/дорожки/
  стерео/диаризация), число спикеров, модель, устройство (авто/CUDA/CPU),
  чекбоксы форматов вывода, лог с прогрессом XXL (парсить его stdout),
  кнопка открытия папки результата.
- Прогресс важен пользователю: показывать этапы и проценты, не молчать.

### Параметры XXL в GUI (референс — рабочая команда пользователя в Subtitle Edit:
`--device cpu --threads 16 --compute_type int8_float32 --language ru --vad_filter true --diarize pyannote_v3.1 --diarize_device cpu`)

Главный экран: язык (ru по умолчанию / en / авто), устройство (Авто/CUDA/CPU),
число спикеров, модель.

Свёрнутая панель «Дополнительно» (все значения сохраняются в settings.json):
- `--threads`: число, по умолчанию «авто» = физические ядра;
- `--compute_type`: по умолчанию «авто» — int8_float32 на CPU, float16 на CUDA;
  вручную: int8 / int8_float32 / float16 / float32;
- `--vad_filter`: галочка, ВКЛЮЧЕНА по умолчанию;
- `--diarize`: движок pyannote_v3.1 (по умолч.) / pyannote_v3.0 / reverb_v1 /
  reverb_v2 — виден только в режиме диаризации;
- `--diarize_device`: «как основное устройство» по умолчанию + переопределение
  (важный кейс: Whisper на CUDA + диаризация на CPU при VRAM 4-6 ГБ);
- поле «Дополнительные аргументы» (свободная строка, дописывается к команде
  XXL как есть) — обязательный escape hatch для остальных флагов.
- рядом с этим полем — кнопка «Справка по параметрам»: открывает встроенное
  окно-просмотрщик (read-only, с полем поиска по тексту) с содержимым файла
  «purfview FastWhisper parameters.txt» (лежит в корне проекта, 407 строк —
  полный список флагов XXL; в сборке хранить рядом с GUI и показывать из него).

## Структура папки

```
Transcriber-Portable\
├── gui\        — наш интерфейс (exe через PyInstaller или embedded Python)
├── xxl\        — Faster-Whisper-XXL как есть
├── ffmpeg\     — статический ffmpeg (gyan.dev essentials)
├── models\     — кеш моделей Whisper и диаризации (едет с папкой)
└── run.bat     — запуск GUI
```

## Порядок реализации (поэтапно, каждый этап — рабочий результат)

1. Скрипт-оркестратор без GUI (CLI): все три режима, проверка на реальной
   записи пользователя. Сверить качество склейки с эталоном пользователя
   (Subtitle Edit + XXL он считает «близким к идеалу»).
2. GUI поверх оркестратора.
3. Упаковка в портативную папку, тест переноса.

## Факты, установленные на этапе 1 (2026-07-18)

- XXL r245.4 распакован в xxl\; свой ffmpeg.exe в комплекте; ffprobe добавлен
  в ffmpeg\ (сборка BtbN).
- `--model_dir` по умолчанию = xxl\_models (внутри папки) — портативно само.
- Модели диаризации (pyannote VAD, WeSpeaker) ВШИТЫ в xxl\_xxl_data\...\assets —
  HF-токен НЕ НУЖЕН вообще, поле токена в GUI не делать.
- CUDA-библиотеки (cuDNN, cuBLAS) в комплекте — один дистрибутив на CPU и GPU.
- Скорость на 3900X (CPU, turbo, int8_float32): распознавание ~0.9x реального
  времени; распознавание+диаризация 7.5 мин файла = 6:42.
- Умолчание XXL — device=cuda: ВСЕГДА явно передавать --device.
- Вывод с --diarize (txt): `[MM:SS.mmm --> MM:SS.mmm]  [SPEAKER_NN]: текст`;
  json содержит сегменты+слова. Качество диаризации фраза-в-фразу совпало
  с эталоном пользователя из Subtitle Edit.

## Известные грабли из прошлого опыта

- Сведение сегментами (по midpoint/перекрытию сегмента) даёт кашу в диалогах —
  только пословное/по готовым репликам XXL. Подробности и алгоритм — в
  «Пайплайн транскрибации с диаризацией.md» рядом с этим файлом.
- PowerShell 5.1 требует UTF-8 BOM для кириллицы в .ps1.
- MKV предпочтительнее MP4 для записи OBS (выживает при обрыве).
- У пользователя: Ryzen 3900X, GT 1030 2GB (только CPU), Windows 11,
  ffmpeg установлен, Python 3.12 есть (но портативная сборка не должна
  от него зависеть).
